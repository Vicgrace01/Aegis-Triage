# Aegis: Protocol-Guided Emergency Stabilization

> **When every second matters and connectivity is unavailable, Aegis turns a frightened bystander into a guided first responder using safe, offline AI.**

Aegis is an offline emergency stabilization system designed for rural and low-resource environments where internet connectivity, cloud services, and immediate professional medical assistance may be unavailable.

Unlike conventional AI assistants, Aegis does **not** generate free-form medical advice. Instead, it combines constrained natural language understanding with deterministic emergency protocols to produce transparent and reproducible stabilization guidance.

---

## Motivation

In many rural communities across Africa, emergency care is delayed by poor connectivity, long transport times, and limited access to trained medical personnel.

During these critical minutes, the first person on the scene is often an untrained bystander.

Aegis bridges this gap by helping users convert a chaotic description of an emergency into protocol-guided stabilization steps that can be executed before professional medical care becomes available.

# System Architecture

```text
                Emergency Description
                         │
                         ▼
             Quantized Qwen2.5-0.5B LLM
          (Natural Language Understanding)
                         │
                         ▼
          Structured Clinical Variables
                 (JSON Output)
                         │
                         ▼
       Deterministic WHO Protocol Engine
                         │
                         ▼
      Emergency Stabilization Instructions
                         │
                         ▼
       Nearest Medical Facility Recommendation
```

The language model is responsible only for extracting structured clinical information.

All emergency recommendations are generated by deterministic protocol logic.

---

# Features

## Protocol-Constrained Reasoning

Custom GBNF grammar constraints restrict the language model to producing only valid structured outputs.

## Offline Operation

Runs entirely on CPU without requiring cloud services or internet connectivity.

## Deterministic Emergency Guidance

Medical recommendations are produced using predefined emergency protocols rather than probabilistic text generation.

## Low Hardware Requirements

Designed for commodity hardware with approximately 600 MB RAM usage using a quantized Qwen2.5-0.5B model.

## Explainable Outputs

Every recommendation can be traced back to the extracted clinical variables and protocol pathway used.

# Installation

Clone the repository.

```bash
git clone https://github.com/<username>/Aegis-Triage.git
```

Enter the project directory.

```bash
cd Aegis-Triage
```

Install the required packages.

```bash
pip install -r requirements.txt
```

Download the quantized GGUF model and place it inside the `models/` directory.

---

# Running Aegis

```bash
python aegis.py
```

Enter an emergency description when prompted.

Example:

```text
The victim fell from a motorcycle. He is bleeding heavily from his leg but is still conscious and breathing.
```

The system extracts structured clinical variables before producing deterministic emergency stabilization instructions.

# Design Philosophy

Aegis separates AI language understanding from medical reasoning.

```
Natural Language
        │
        ▼
Small Language Model
        │
        ▼
Structured Symptoms
        │
        ▼
Deterministic Protocol Engine
        │
        ▼
Emergency Instructions
```

This architecture improves transparency, reproducibility, and computational efficiency while reducing reliance on unrestricted language generation.
